# Next Phase Repeatability Plan v1

Purpose: define what should happen next after the pipeline freeze.

## Guiding decision

The next phase should focus on **repeatability**, not reinvention.

Reason:
- the engine path is stable enough
- the authoring path is now defined
- the design layer exists
- one generated lesson and one upgraded lesson already exist

What is still missing is evidence that the frozen pipeline works repeatedly across multiple fresh lessons.

## Next phase objective

Generate and run additional lessons through the frozen pipeline, then observe what repeats.

## Priority order

### Priority 1: repeated lesson generation
Create at least two more fresh lessons using the same path:
- teacher brief
- package generation
- render + QA
- optional engagement upgrade

Recommended targets:
- one more single-period lesson in a different subject
- one multi-day sequence lesson

### Priority 2: repeated comparison
For at least one of those lessons, compare:
- baseline version
- upgraded engaging version

Use the baseline-to-engaging rubric.

### Priority 3: friction log
Track repeated friction points such as:
- package generation ambiguity
- awkward CLI workflow for generated lessons
- repeated worksheet weaknesses
- repeated slide-rhythm weaknesses
- renderer limits that block worthwhile design improvements

## What should not happen yet

Do not jump immediately to:
- major schema rewrites
- major renderer rewrites
- large template/theme systems
- broad content scaling without repeated lesson evidence

## The main question for the next phase

What problems repeat often enough that they deserve first-class support?

That should drive the next investments.

## Expected outputs of the next phase

1. two or more new lesson packages
2. rendered lesson bundles for those packages
3. at least one baseline vs upgraded comparison
4. a short repeated-friction summary
5. a recommendation for the next engine/design investment

## Recommended first concrete target

After the Careers 8 work, the strongest next target would be:
- one single-period lesson in a different subject area, or
- one multi-day lesson that tests continuity/checkpoint/final evidence logic again

## Success rule

The next phase is successful if it shows:
- the frozen pipeline works more than once
- the same kinds of issues repeat clearly enough to guide the next build phase
