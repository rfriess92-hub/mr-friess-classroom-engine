# Repo Documentation Audit — 2026-04-12

## Purpose

This note records the repo-truth mismatches found during the stable-core hardening pass.

## Mismatches found

### 1. `SETUP_STATUS.md` is stale

The file describes the repo as if the real builders, schema files, and complete package/toolchain wiring are not loaded yet.

That no longer matches the actual repo state. The stable-core path already includes:

- schema files
- route planning
- package rendering
- bundle QA
- public renderer entrypoints

Use `docs/current-repo-status.md` as the accurate status snapshot until the legacy file is replaced or rewritten.

### 2. `README.md` legacy command block is too strong for the repo state

The README currently documents legacy compatibility commands as if they are operator-ready CLI surfaces. The stable-core package path is the real acceptance surface.

Legacy direct-builder surfaces should either:

- be documented as incomplete transitional/debugging surfaces, or
- be fully wired and tested before being presented as supported commands

### 3. Workflow guide needs CI wording refresh

The stable-core workflow guide still frames the repo as having no CI mirror at all. After the addition of `.github/workflows/stable-core.yml`, the accurate wording is:

- local execution remains authoritative
- CI mirrors the stable-core contract
- the mirror should be trusted more as it proves out over time

## Follow-through target

When existing docs are next editable through the connector/tooling path, update these files first:

1. `README.md`
2. `SETUP_STATUS.md`
3. `docs/stable-core-workflow-policy.md`
4. `DECISIONS.md`
