# Psychology 11/12 Assessment Spine

## 1. Purpose

This document defines the repeatable assessment structure for Psychology 11/12. It translates the lens-based course architecture into a stable assessment rhythm that can be reused across Cycles A-F without forcing every cycle to use identical tasks.

The assessment spine should protect three priorities:

1. Academic psychology: students define concepts accurately, evaluate claims, and use evidence carefully.
2. Classroom safety: students are not asked to diagnose, disclose private experiences, counsel peers, or analyze classmates.
3. Transfer: students repeatedly apply the same reasoning habits to different psychology lenses.

This is a planning-source document. It does not contain final quizzes, tests, answer keys, or full rubrics.

## 2. Assessment Philosophy

Psychology 11/12 assessment should measure disciplined psychological thinking, not personal confession or opinion performance. Students should be assessed on how well they define, explain, apply, analyze evidence, recognize limits, and use ethical language.

Across all cycles, assessment should reward careful claims over dramatic claims. A strong response says what psychology can reasonably explain, what evidence supports that explanation, and what cannot be concluded.

## 3. Whole-Course Assessment Layers

| Layer | Purpose | Typical Frequency | Student-Facing? | Teacher-Facing Support |
|---|---|---:|---|---|
| Vocabulary and Concept Checks | Build accurate disciplinary language. | Weekly or per module. | Yes. | Answer key and misconception guide. |
| Scenario Applications | Apply psychology concepts to fictional, neutral, or public examples. | Every module. | Yes. | Look-fors and sample responses. |
| Source / Evidence Analysis | Distinguish claim, evidence, inference, opinion, correlation, causation, and limits. | At least twice per cycle. | Yes. | Source notes, answer key, marking guide. |
| Short Quizzes / Tests | Check foundational knowledge and concept accuracy. | One or two per cycle. | Yes. | Teacher-only answer key. |
| Applied Mini-Project | Use psychology responsibly in a realistic setting. | Once per cycle where useful. | Yes. | Rubric and accommodation notes. |
| Capstone / Performance Task | Synthesize multiple domains through the cycle lens. | Once per cycle. | Yes. | Marking guide, safety notes, exemplars. |
| Reflection / Exit Ticket | Check transfer, uncertainty, and responsible language. | Regularly. | Yes. | Optional teacher review notes. |

## 4. Repeating Student Reasoning Pattern

Every substantial assessment should ask students to move through a recognizable reasoning chain:

| Step | Student Action | Assessment Evidence |
|---|---|---|
| 1. Define | Use accurate psychology vocabulary. | Clear concept definition in student language. |
| 2. Explain | Describe the psychological process, theory, or claim. | Accurate explanation without overclaiming. |
| 3. Apply | Connect the concept to a fictional, neutral, or public example. | Relevant application to scenario evidence. |
| 4. Check Evidence | Identify what evidence supports the claim. | Reference to observation, data, source, or method. |
| 5. Name Limits | State what cannot be concluded. | Recognition of uncertainty, correlation/causation limits, sample limits, or context. |
| 6. Use Responsibly | Consider ethics, context, culture, development, and safety. | Careful language and appropriate boundaries. |

## 5. Core Rubric Categories

These categories should appear, with cycle-specific wording, in major assignments and capstones.

| Category | Looks For | Common Drift To Avoid |
|---|---|---|
| Concept Accuracy | Correct terminology, clear definitions, accurate explanation. | Vague advice, pop psychology, unsupported labels. |
| Evidence and Limits | Distinguishes claim, evidence, inference, opinion, correlation, causation, and limitation. | Treating one example as proof; causal overclaiming. |
| Application | Applies concepts to the provided scenario or source. | Personal disclosure, diagnosing, or irrelevant summary. |
| Through-Line Integration | Includes culture/context, development, brain-body, and ethics/evidence where appropriate. | Treating one group, age, or biological explanation as universal. |
| Responsible Language | Avoids diagnosis, peer analysis, stigma, sensationalism, and certainty beyond evidence. | Labelling people, moral panic, true-crime entertainment, therapeutic role-play. |
| Communication | Organized, readable, specific, and appropriate to task format. | Overlong, under-supported, or performative responses. |

## 6. Cycle-Specific Assessment Emphasis

| Cycle | Lens | Assessment Emphasis | Capstone Assessment Frame |
|---|---|---|---|
| A | Scientific / Evidence | Claim analysis, research methods, ethics, correlation/causation, evidence quality. | Evaluate a psychological claim and write a careful conclusion. |
| B | Intellectual / Learning | Learning theory, memory, cognition, motivation, practice, development, context. | Design an evidence-based learning or performance support. |
| C | Relational / Identity | Social cognition, belonging, identity, personality, sexuality, culture, consent, disclosure control. | Analyze a fictional relationship, media scene, or public scenario without diagnosis. |
| D | Wellness / Change | Stress, coping, mental health literacy, support systems, treatment awareness, stigma reduction. | Create a support-awareness product or analyze a fictional case with clear boundaries. |
| E | Justice / Responsibility | Eyewitness memory, bias, authority, responsibility, development, forensic ethics. | Analyze a sanitized fictional legal/forensic case file. |
| F | Systems / Media / Influence | Attention, persuasion, behaviour design, misinformation, consumer psychology, AI, ethical design. | Conduct a media, technology, advertising, gaming, or consumer-system audit. |

## 7. Assessment Artifact Model

Each cycle should eventually include the following assessment artifacts.

| Artifact | Audience | Contents | Separation Rule |
|---|---|---|---|
| Student Assessment Pack | Student | Quizzes, application tasks, mini-project task, capstone instructions, student-facing rubric. | No answer keys, teacher notes, or hidden marking rationale. |
| Teacher Marking Guide | Teacher | Answer keys, look-fors, sample responses, rubric notes, accommodations, misconception notes. | Must be clearly teacher-facing. |
| Capstone Packet | Student | Scenario/source packet, task instructions, success criteria, submission structure. | Uses fictional, neutral, public, or source-based material. |
| Source / Evidence Sheet | Student | Short source excerpts or paraphrased evidence supports. | Traceable but not a textbook reproduction. |
| QA Checklist | Teacher/repo | Visibility, safety, answer-leakage, disclosure, and artifact inventory checks. | Confirms student-facing and teacher-facing boundaries. |

## 8. Student-Facing Assessment Rules

Student-facing assessments must not require students to:

- disclose trauma, diagnoses, family history, private relationships, sexuality, identity struggles, therapy history, medication use, or personal distress
- diagnose themselves, peers, teachers, family members, public figures, or fictional characters as if they were clinicians
- role-play therapy, crisis counselling, forensic profiling, or peer support beyond student boundaries
- rank classmates by intelligence, personality, attachment style, mental health, relationship skill, or vulnerability
- turn legal psychology into graphic true-crime entertainment
- shame technology use, media habits, identity expression, or family/community background

Use instead:

- fictional scenarios
- sanitized case files
- neutral school/work/community examples
- public media examples with careful boundaries
- source analysis
- low-risk academic reflection

## 9. Teacher-Facing Assessment Supports

Teacher-facing assessment supports may include:

- answer keys
- model responses
- misconception notes
- safety notes
- differentiation and accommodation guidance
- scoring guidance
- discussion cautions
- source notes
- feedback language

These materials must remain clearly separated from student-facing assessment files.

## 10. Cycle Safety Modifiers

| Cycle | Safety Modifier |
|---|---|
| A | Watch for overclaiming, unsupported research conclusions, and pop-psychology drift. |
| B | Avoid fixed-ability framing, public ranking, manipulative behaviour-control framing, and deficit language. |
| C | Avoid personal relationship disclosure, attachment-style sharing, forced identity disclosure, and public personality sorting. |
| D | Avoid diagnosis, trauma disclosure, therapy simulation, crisis role-play, and peer counselling. |
| E | Avoid graphic crime details, real-person diagnosis, amateur profiling, and sensationalism. |
| F | Avoid moral panic, student shaming, manipulative-design assignments, and forced disclosure of online habits. |

## 11. Minimal Rubric Scale

Use a simple four-level structure for major assessments unless a specific task requires otherwise.

| Level | Descriptor | Meaning |
|---:|---|---|
| 4 | Strong / Transferable | Accurate, evidence-aware, responsible, and able to transfer across context. |
| 3 | Proficient | Mostly accurate and appropriate; may need more detail or clearer limits. |
| 2 | Developing | Partial understanding; some vague, unsupported, or incomplete reasoning. |
| 1 | Beginning | Limited accuracy, weak evidence use, unsafe overclaiming, or major misunderstanding. |

Avoid using rubric language that implies students are psychologically flawed. Assess the work, not the student.

## 12. Capstone Design Template

Each cycle capstone should include:

| Component | Requirement |
|---|---|
| Lens Anchor | Names the cycle lens and core question. |
| Scenario / Source Base | Uses fictional, neutral, public, sanitized, or source-based material. |
| Concept Set | Requires multiple psychology domains, not one isolated term. |
| Evidence Requirement | Requires students to identify support and limits. |
| Through-Line Requirement | Includes culture/context, development, brain-body, and ethics/evidence where appropriate. |
| Safety Boundary | States what students should not diagnose, disclose, or conclude. |
| Student Rubric | Uses student-facing success criteria. |
| Teacher Marking Guide | Separate teacher-facing scoring support. |

## 13. Feedback Language

Feedback should reinforce disciplinary thinking.

Use feedback such as:

- Your concept definition is accurate; now add the evidence limit.
- This is a reasonable application, but avoid turning the scenario into a diagnosis.
- You identified a possible explanation; now state what cannot be concluded.
- Your evidence is relevant, but the claim needs more careful language.
- Strong connection to context; add the developmental or brain-body angle if relevant.

Avoid feedback such as:

- You are anxious / avoidant / biased / manipulative.
- This proves the character has a disorder.
- You should use this on your friends.
- This is how people always behave.
- This app / relationship / behaviour is toxic.

## 14. Assessment Build Order

Build assessment materials in this order:

1. Course-level assessment spine.
2. Student-facing capstone frame for Cycle A.
3. Teacher-facing Cycle A marking guide.
4. Cycle A vocabulary and concept checks.
5. Cycle A source/evidence analysis tasks.
6. Repeat assessment pack structure for Cycles B-D.
7. Keep Cycles E-F mapped until active source files exist.

## 15. QA Checklist For Assessment Packs

Before any assessment pack is considered ready:

- Student-facing documents contain no answer keys or teacher-only rationale.
- Teacher-facing documents are clearly labelled.
- No assessment requires private disclosure.
- No task asks students to diagnose or counsel.
- Each major task includes evidence and limits.
- Each capstone includes more than one psychology domain.
- Cycle-specific safety controls are visible.
- Rubric language assesses work, not student identity or psychology.
- OpenStax concepts are used as the academic spine without reproducing textbook material directly.
