# Printable Package System Rules

## Purpose

Use this file to keep the printable packets aligned.

The current packet set includes the Launch Packet, Voice Trial, Word Vault, and Insight Gate. These documents should feel like one system, not four separate systems.

## Core rule

Reading comes first.

The game layer is there to make practice visible, safe, and easier to talk about.

## Strategies and power-ups are different

### Strategies

Strategies are plain reading moves students can use when stuck.

Students do not earn strategies. They can use them any time.

Examples:

- reread a smaller part
- look back at the text
- ask to hear the first line
- find a clue
- fix one mistake and keep going

### Power-ups

Power-ups are character supports with limited uses during a quest.

They make help visible and trackable.

They are not prizes, and they are not reading levels.

Most power-ups should have 3 uses per quest.

### Help

Basic help is always available.

A student should never be blocked from needed help because they do not have the right power-up.

## Character model

Students have one core character.

Each arc gives that character an arc support kit.

Do not make students feel like they are choosing a new character every packet.

Use this wording:

`Your character gets a support kit for this arc.`

Avoid this wording:

`Choose a new character for this arc.`

## Level-up rule

Leveling up gives students more choice or control.

It does not decide who deserves help.

Good level-up choices:

- add one use to a power-up
- swap one support before the next quest
- use one support more independently
- help someone choose a stuck-list move

Avoid level-ups that give stronger readers more help while weaker readers have less.

## Student-facing tone

Use direct classroom language.

Tone model:

`Your character is a way to track how you practise, what strategies you try, and what gets stronger.`

Avoid product language, sales language, and long explanations of the whole system.

Students need the move for today, not the full theory behind the program.

## Printing rule

Do not hand students the whole package at once.

Treat each packet as a page bank.

Print only:

- the current student intro if needed
- the current quest page
- the stuck list
- the power-up cards students need
- the boss page when ready

Keep teacher setup pages and trackers separate.

## Accessibility standard

Use the current formatting standard:

- Calibri
- 14 pt student text
- 12 pt teacher text
- 1.2 line spacing
- left aligned text
- wide answer lines, not boxed response areas
- short paragraphs
- limited game terms per page
