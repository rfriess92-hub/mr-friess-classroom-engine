# Teacher Launch Checklist

## Purpose

This checklist helps the Game Master launch the Literacy Guild without overloading students or the teacher.

The launch goal is simple: students understand the three paths, the core loop, the dignity rules, and how to submit first evidence.

## Core launch rule

Routines first. Mechanics second. Lore third.

If students are confused, simplify the game layer and protect the reading routine.

## Before the first class

Prepare these materials:

- student overview
- Guild Passport
- character sheet
- teacher tracker
- quest completion tracker
- Game Master reference card
- Voice / Insight / Wordcraft display
- short shared text
- first mini quest packet
- exit tickets or reflection slips

## Room setup

Set up:

- teacher table area
- party work areas
- independent work area
- visible daily agenda
- visible three-path display
- place for private teacher materials

Do not post public progress charts that compare students.

## Week 1 launch priorities

By the end of Week 1, students should know:

- this is a reading training course
- Voice means fluency
- Insight means comprehension
- Wordcraft means vocabulary
- the class uses the core loop
- evidence matters more than XP
- progress information is private
- party work helps but does not replace solo evidence

## Week 2 launch priorities

By the end of Week 2, students should:

- start or refine a character sheet
- use first ability cards
- try simple party roles
- understand that quest choice is controlled choice
- submit a short solo evidence task
- log XP or reflection without too much teacher time

## First mini quest readiness

Run the first mini quest only when students can:

- identify the three paths with support
- complete a short Character Log
- work in a party without losing the task
- understand that final evidence must be individual

If not ready, repeat a smaller training task first.

## Materials release order

Do not hand out everything at once.

Recommended order:

1. Student overview
2. Guild Passport
3. Three-path display
4. Character sheet
5. First three ability cards
6. First three party role cards
7. Student quest map
8. First mini quest packet

## First ability cards to use

Start with:

- Echo Cast
- Gist Map
- Context Torch

Do not introduce the full ability deck until routines are stable.

## First party roles to use

Start with:

- Quest Captain
- Insight Keeper
- Wordsmith

Add more roles after students can handle basic party work.

## First XP guidance

Keep XP simple.

Use only a few options:

- 10 XP for training task
- 15 XP for using a named strategy
- 20 XP for clear solo evidence
- 10 XP for reflection

Award in batches. Do not track every small action.

## First evidence expectations

Early evidence can be short.

Acceptable first evidence:

- one gist statement
- one revised gist statement
- one context clue explanation
- one reread reflection
- one short answer with a text detail

The evidence must be understandable, not perfect.

## Teacher tracker setup

Track only what matters at launch:

- current path goal
- evidence submitted
- support needed
- first ability used
- next teacher-table move

Delay detailed tracking until students understand the system.

## Progress check caution

Use baseline and progress checks privately.

Do not connect scores directly to public character level, rank, or party status.

Progress checks should help choose instruction, not label students.

## Student language to repeat

Use these lines often:

- Your character is not your reading level.
- Party work helps you train, but your evidence is your own.
- A reroute is not failure.
- XP rewards training, but evidence shows growth.
- You can choose the route, but everyone trains all three paths.

## Game Master reset moves

Use these when the class drifts:

| Problem | Reset move |
|---|---|
| Students chase points | Pause XP and return to evidence. |
| Students compare progress | Restate privacy rule and remove public comparison. |
| Party work gets noisy | Reduce roles and return to solo evidence. |
| Mechanics take too long | Drop the mechanic for the day. |
| Students do not understand paths | Reteach Voice, Insight, and Wordcraft with examples. |

## Launch success criteria

The launch is working if:

- students can name the path they are training
- students can submit one piece of solo evidence
- students understand that progress is private
- parties support rather than replace individual work
- the teacher can track next moves quickly
- the game layer feels mature and useful

## Launch guardrail

Do not rush into a full path arc until the basic routine works.

The class is ready for the first larger arc when students can complete the core loop with limited prompting.
