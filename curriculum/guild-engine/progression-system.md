# Guild Engine Progression System

## Purpose

This file defines the concrete progression rules for the Guild Engine.

The system uses three connected layers:

1. XP for training behaviours.
2. Ranks for demonstrated skill evidence.
3. Levels for overall course campaign progress.

## Design rule

XP rewards training. Rank-ups require evidence. Levels mark campaign growth.

A level is never an academic level, ability group, reading level, or public comparison tool.

## XP principles

XP should be:

- fast to award
- easy to explain
- tied to learning actions
- small enough to manage
- flexible across subjects

XP should not require the teacher to do constant bookkeeping during instruction.

## Standard XP menu

Use this as the default menu. Course versions may rename the categories, but should keep the logic.

| XP Type | Suggested XP | Evidence |
|---|---:|---|
| Training XP | 10 | Completed useful practice |
| Strategy XP | 15 | Used a named strategy correctly |
| Feedback XP | 15 | Improved after feedback |
| Evidence XP | 20 | Submitted clear evidence of skill |
| Reflection XP | 10 | Named what changed and what comes next |
| Growth XP | 20 | Improved from personal baseline |
| Support XP | 10 | Helped a peer use a strategy without doing the work |

## XP limits

To keep the system manageable:

- Award XP in batches when possible.
- Do not score every tiny action.
- Do not use XP as a behaviour economy.
- Do not subtract XP as punishment.
- Do not let XP replace feedback.
- Do not let XP become more important than evidence.

## Rank structure

Each course skin should define three learning paths.

Each path should have five ranks.

Recommended rank pattern:

| Rank | Meaning |
|---:|---|
| 1 | I can attempt the skill with support. |
| 2 | I can use the skill in a simple task. |
| 3 | I can use the skill with more independence. |
| 4 | I can apply the skill in a larger task. |
| 5 | I can explain or transfer the skill. |

## Rank-up requirements

A rank-up requires:

1. named skill
2. evidence standard
3. student evidence
4. teacher confirmation
5. next goal

A rank-up should not be automatic from XP alone.

## Evidence standards

Evidence should be:

- concrete
- observable
- connected to the path
- understandable to the student
- reasonable to check quickly

Evidence can be:

- written work
- oral explanation
- recording
- conference note
- revised task
- teacher observation
- completed reflection

## Level structure

Levels show broad campaign progress. They are lighter than ranks.

Suggested level bands:

| Level | Title | Requirement |
|---:|---|---|
| 1 | Initiate | Character created and first quest started |
| 2 | Apprentice | 100 XP and at least one evidence entry |
| 3 | Explorer | 250 XP and at least one rank attempt |
| 4 | Specialist | 450 XP and one confirmed rank-up |
| 5 | Guild Member | 700 XP and one boss task completed |
| 6 | Adept | 1000 XP and evidence in all three paths |

Course versions can adjust XP totals, but should preserve the idea that levels mark broad campaign progress.

## Badges and checkpoints

Badges can mark demonstrated strategies. Use them sparingly.

Good badge examples:

- Strategy Used
- Strong Revision
- Evidence Builder
- Helpful Support
- Clear Reflection

Avoid making a large badge catalogue before the basic system works.

## Progression privacy

XP totals and levels may be visible if they do not create harmful comparison.

Rank evidence, assessment results, and individual goals should be handled carefully and usually kept between teacher and student.

## Teacher workload rule

The progression system is too heavy if:

- the teacher spends more time tracking than coaching
- students argue about small point values
- students chase XP without improving work
- ranks are awarded without clear evidence
- public progress creates comparison problems

If this happens, reduce point categories, award weekly instead of daily, or remove public totals.

## Minimum classroom version

A launch-ready progression system needs only:

- one XP menu
- one level chart
- one rank ladder per learning path
- one evidence log
- one teacher confirmation routine
- one reflection prompt
