# Reusable Class Model

## Principle

The Guild Engine should separate the game structure from the subject content.

## Layer 1: Engine

Reusable across classes:

- quest
- training task
- evidence log
- XP
- rank
- level
- boss task
- reflection
- conference
- class milestone

## Layer 2: Course skin

Each class defines its own learning paths.

## Example skins

### Literacy Guild

- Voice
- Insight
- Wordcraft

### Science Guild

- Observe
- Explain
- Apply

### Social Studies Guild

- Evidence
- Perspective
- Cause and Consequence

### Writing Guild

- Ideas
- Structure
- Style

### Design Guild

- Plan
- Build
- Improve

## Transfer rule

When adapting the model, keep the same loop and replace only the learning paths, evidence types, and quest content.
