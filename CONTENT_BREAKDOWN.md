This document gives the full content breakdown for the Psychology 11/12 classroom package and render repo.

- course structure

## 3. Source Spine

The core academic spine is:

OpenStax Psychology 2e

OpenStax Psychology 2e provides the core concepts, vocabulary, chapter structure, and disciplinary grounding.

The classroom package adapts that spine into:

- high school classroom lessons
- student packets
- teacher binders
- assessment tasks
- discussion prompts
- source-based activities
- scenario-based application
- capstone work
- safe reflection activities

The package should not reproduce the textbook directly. It should translate the textbook into usable classroom materials.

## 4. Course Identity

Course:

Psychology 11/12

Intended classroom use:

Senior secondary psychology course

Instructional frame:

Concept-based, classroom-safe, source-supported, evidence-focused psychology instruction

Core classroom stance:

Students learn to think like psychology students, not diagnose themselves or others.

---

## 5. Core Course Principles

The course should repeatedly teach students to:

- define psychological concepts accurately
- distinguish evidence from opinion
- avoid jumping to conclusions
- interpret behaviour carefully
- recognize uncertainty
- apply concepts to neutral scenarios
- use research and source material responsibly
- analyze claims about behaviour and mental processes
- avoid self-diagnosis and amateur diagnosis
- discuss mental health with safety and precision

The course should avoid:

- therapy-style personal disclosure
- trauma-mining prompts
- diagnostic tasks directed at students
- oversimplified pop psychology
- “tell your story” activities
- unsupported claims about behaviour
- content that makes students responsible for fixing others

The full course system is organized into six cycles.

```text
Cycle A: Foundations
Cycle B: Learning / Cognition / Development / Applied Psychology
Cycle C: Personality / Identity / Social Psychology / Relationships
Cycle D: Mental Health / Stress / Disorders / Treatment / Support Systems
Cycle E: Forensic and Legal Psychology

```

Cycles A–D are the active built classroom package.

Cycles E–F are mapped in the three-year cycle model and should be treated as future or extension build areas unless active files are explicitly present.

## 8. Standard Cycle Architecture

Each cycle should eventually contain the same major components.

```text
Teacher Binder
Student Packet
Assessment Pack
Source Sheet
Capstone Packet
Slide Deck
QA Bundle
```

### Teacher Binder

Teacher-facing delivery guide.

Includes:

- cycle overview
- lesson sequence
- timing
- teacher notes
- safety notes
- discussion guidance
- misconceptions
- differentiation
- assessment guidance
- answer keys where relevant

### Student Packet

Student-facing classroom workbook.

Includes:

- learning targets
- vocabulary
- activity instructions
- source excerpts
- guided practice
- reflection prompts
- exit tickets
- assessment preparation

Must not include:

- answer keys
- teacher rationale
- marking notes
- hidden implementation notes

### Assessment Pack

Assessment-facing materials.

Should include:

- student assessment version
- teacher assessment version
- rubric
- answer key
- marking guide
- feedback language
- accommodation notes

### Source Sheet

Approved source-facing support.

May include:

- OpenStax excerpts
- classroom-safe source excerpts
- source context
- reading prompts
- evidence questions
- vocabulary support

### Capstone Packet

Larger synthesis task.

May include:

- scenario analysis
- research critique
- applied psychology explanation
- source-supported argument
- claim-evidence-reasoning task
- presentation or written response

### Slide Deck

Student-facing visual delivery.

Should include:

- low-density slides
- lesson targets
- task instructions
- concept explanations
- checks for understanding
- discussion prompts
- takeaways

Teacher notes must remain hidden from student-facing slides.

### QA Bundle

Render and review support.

Should include:

- manifest
- known issues
- render warnings
- answer-leakage check
- artifact checklist
- screenshots or previews where possible

## 20. Learning Cycle Model

Each classroom lesson should generally follow this learning cycle.

```text
1. Activate
2. Teach
3. Apply
4. Check
5. Reflect
6. Transfer
```

### Activate

Purpose:

Connect to prior knowledge or surface assumptions.

Examples:

- quick question
- misconception prompt
- scenario
- image or claim analysis
- retrieval practice

### Teach

Purpose:

Introduce the core concept clearly.

Examples:

- mini-lesson
- vocabulary explanation
- short source excerpt
- teacher model
- concept comparison

### Apply

Purpose:

Use the concept on a task.

Examples:

- scenario analysis
- sort activity
- claim-evidence-reasoning
- source annotation
- partner task
- case comparison

### Check

Purpose:

Confirm understanding before moving on.

Examples:

- multiple choice check
- short answer
- exit ticket
- misconception correction
- explain-your-choice task

### Reflect

Purpose:

Consolidate learning safely.

Examples:

- academic reflection
- strategy reflection
- concept reflection
- “what changed in your thinking?”

Avoid personal disclosure.

### Transfer

Purpose:

Apply learning to a new situation.

Examples:

- new scenario
- media example
- assessment prep
- capstone connection

## 22. Differentiation Model

Use the same three tiers across the package.

```text
Supported
Proficient
Extending
```

### Supported

Students receive:

- fewer examples
- clearer sentence frames
- chunked instructions
- vocabulary support
- guided choices
- reduced writing load

### Proficient

Students complete:

- standard task
- standard explanation
- standard vocabulary use
- standard classroom product

### Extending

Students complete:

- transfer task
- original example
- deeper comparison
- stronger evidence use
- more independent reasoning

The learning target stays the same across tiers.

## 23. Safety Model

The package must preserve classroom safety.

### General Safety Rules

Do not require students to:

- disclose trauma
- disclose diagnoses
- analyze their own mental health publicly
- diagnose themselves
- diagnose peers
- share family history
- role-play therapy
- provide counselling
- reveal private relationships
- expose personal identity struggles

### Preferred Alternatives

Use:

- fictional scenarios
- public examples
- neutral case studies
- low-risk reflection
- concept application
- source analysis
- evidence evaluation

### Mental Health Rule

Mental health lessons should teach literacy, stigma reduction, support awareness, and careful language.

They should not become therapy or self-diagnosis lessons.

## 33. Final Content Rule

This package should be built around classroom reality.

A content item is ready only when it is:

- accurate
- safe
- student-readable
- teacher-usable
- clearly tagged
- connected to the course spine
- separated by audience
- ready for rendering
- not dependent on hidden explanation

The renderer should not have to guess what content is for students, what content is for teachers, or what content contains answers.

If the repo preserves that separation, the package can scale.
