# Handoff Constraints

These are non-negotiable pedagogy constraints for rendering.

## Non-negotiables

- teacher/student separation is non-negotiable
- output-role integrity is non-negotiable
- bundle integrity is non-negotiable
- preserve pedagogy meaning, not just content presence
- curriculum visibility stays teacher-facing by default
- embedded support elements are not standalone outputs by default
- final evidence must remain in its declared location
- multi-day packages must preserve day boundaries, checkpoints, and carryover logic

## Renderer rules

The renderer must not:

- invent undeclared outputs
- merge outputs aggressively by default
- duplicate final evidence unless explicitly declared
- move teacher notes into student-facing artifacts
- flatten multi-day structure into a single undifferentiated output set

## Hybrid rule

A package may declare one `primary_architecture` and one `secondary_architecture_support`.
The secondary architecture must remain subordinate and must not override the primary output bundle or audience rules.
