# Slide Pattern Library v1

Purpose: define reusable slide patterns that improve classroom rhythm while staying compatible with the stable-core lesson package model.

## Design rule

A strong deck should not feel like:
- title
- prompt
- bullets
- more bullets
- reflect

Instead, a strong deck should alternate between:
- launch
- model
- compare
- classify
- apply
- reflect

## Recommended slide patterns

### `launch_prompt`
Use when students need an accessible entry point.

Best for:
- activating prior knowledge
- surfacing assumptions
- low-stakes opening talk

### `visual_compare`
Use when students need to compare examples, pathways, cases, or representations.

Best for:
- compare/contrast
- this vs that
- example vs non-example

### `card_sort_intro`
Use when students are preparing for a classification task.

Best for:
- cluster/category lessons
- sorting logic
- criteria-setting

### `ranking_continuum`
Use when students should place ideas or choices on a spectrum.

Best for:
- opinion with evidence
- strongest to weakest evidence
- best fit / least fit decisions

### `case_study_decision`
Use when students need to read a short scenario and decide what fits best.

Best for:
- careers
- inquiry
- ethical reasoning
- application of concepts

### `worked_example`
Use when the lesson benefits from teacher modeling or a shared sample.

Best for:
- procedural tasks
- analytical reasoning
- justification practice

### `guided_sort`
Use when students need shared practice before independent work.

Best for:
- classification
- matching
- evidence sorting

### `task_launch`
Use when students need to shift from listening to doing.

Best for:
- worksheet launch
- group work launch
- station launch

### `reflection_exit`
Use when students need to consolidate learning and prepare for the final evidence artifact.

Best for:
- exit tickets
- self-assessment
- metacognitive closure

## Pattern guidance by engagement mode

### `sort_classify`

- `launch_prompt`

- `reflection_exit`

Recommended sequence:
- `launch_prompt`

Recommended sequence:
- `launch_prompt`

- `reflection_exit`

## Constraints

The slide pattern library must not:
- push teacher-only guidance into student-visible slides
- introduce final evidence placement confusion
- overwhelm the deck with too many layout changes
- add novelty without instructional purpose

## Minimum rule for v1

Every deck should include at least:
- one real entry/launch pattern
- one structured thinking pattern
- one task-launch pattern
- one reflection/closure pattern
